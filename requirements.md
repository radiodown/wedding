# 모바일 청첩장 프로젝트 기획서

## 1. 프로젝트 개요

GitHub Pages 기반의 **모바일 청첩장 웹사이트**를 제작한다.
개발은 AI 보조 코딩 방식을 적극 활용하며, 유지보수가 쉽고 배포가 간단한 **정적 프론트엔드 구조**를 목표로 한다.

이 프로젝트는 단순한 안내 페이지가 아니라, **사진·영상·오시는 길·연락/공유 기능**까지 포함한 완성도 높은 모바일 경험을 제공하는 것을 목표로 한다.

---


## 2. 개발 목표

* 모바일에서 보기 좋은 **반응형 청첩장 페이지** 제작
* GitHub Pages에 배포 가능한 **정적 웹사이트 구조** 사용
* **카카오지도 API**를 이용한 예식장 위치 안내
* 사진첩/갤러리를 통해 사진을 보기 좋게 표시
* **YouTube 링크 또는 임베드 영상** 제공
* AI를 활용해 빠르게 개발하고, 이후 수정도 쉬운 구조로 설계
* config를 사용해 색상, 테마 등을 쉬게 보정 할 수 있도록 설계
* 웨딩감성이면서, 모던한 느낌의 디자인

## 4. 기술 스택 선정 이유

### React

* 섹션별 컴포넌트 분리가 쉬움
* AI에게 컴포넌트 단위로 코드 생성을 요청하기 좋음
* 유지보수와 재사용성이 좋음

### Vite

* 개발 서버가 빠름
* GitHub Pages 배포용 빌드가 간단함
* React 프로젝트 시작이 가벼움

### TypeScript

* AI가 작성한 코드의 오류를 줄이는 데 유리함
* props, 데이터 구조, API 응답 형태를 명확히 관리 가능

### Tailwind CSS

* 모바일 퍼스트 UI 개발이 빠름
* 여백, 폰트, 카드, 버튼, 섹션 스타일링이 편함
* 디자인 수정이 빠르고 일관성 유지가 쉬움

## 5. 필수 요구사항

### 5.1 카카오지도 API 사용

* 예식장 위치를 지도에 표시해야 함
* 주소, 예식장명, 교통편 정보를 함께 제공해야 함
* 지도 하단 또는 별도 섹션에 다음 정보 제공 권장

  * 도로명 주소
  * 지하철/버스 안내
  * 주차 안내
  * 카카오맵/네이버지도/티맵 길찾기 외부 링크

### 5.2 사진첩 게시

* 신랑/신부 사진 또는 웨딩 사진을 여러 장 업로드할 수 있어야 함
* 사진은 모바일 환경에서 부드럽게 스크롤되거나 격자형으로 표시되어야 함
* 이미지가 많은 경우에도 로딩 부담이 적어야 함

### 5.3 사진을 예쁘게 보여주는 UI

* 썸네일 리스트 + 클릭 시 확대 보기 지원
* 이미지 비율이 제각각이어도 레이아웃이 깨지지 않아야 함
* 추천 방식

  * masonry 스타일 갤러리
  * 카드형 2열/1열 반응형 그리드
  * 라이트박스 확대 보기
  * 부드러운 전환 애니메이션

### 5.4 YouTube 링크 필요

* 축하 영상, 웨딩 영상, 프러포즈 영상 등을 연결할 수 있어야 함
* 단순 링크뿐 아니라 **임베드 플레이어** 지원 권장
* 모바일에서 너무 크지 않게 비율(16:9) 유지 필요

### 5.5 모바일 반응형 대응

* 스마트폰 화면 폭에 맞춰 자동 레이아웃 조정
* 작은 화면에서도 글씨, 버튼, 사진이 보기 좋아야 함
* 세로 스크롤 중심의 모바일 퍼스트 설계 적용
* 주요 테스트 기준

  * iPhone 계열
  * Galaxy 계열
  * 모바일 브라우저 Safari / Chrome

## 6. 추천 추가 요구사항

### 6.1 첫 화면(히어로 섹션)

* 커버 이미지 또는 대표 사진
* 신랑/신부 이름
* 예식 일시
* 간단한 초대 문구
* 아래로 스크롤 유도 요소

### 6.2 일정 안내

* 결혼식 날짜와 시간
* 요일 포함 표기
* 디데이(D-Day) 표시 선택 가능

### 6.3 연락 기능

* 혼주/신랑/신부 연락처 표시
* 전화 걸기 링크 제공
* 계좌번호 복사 기능이 필요하다면 버튼 제공

### 6.4 공유 기능

* 링크 복사 버튼
* 카카오톡 공유 대응 검토
* 오픈그래프(OG) 메타 태그 설정으로 공유 썸네일 최적화

### 6.5 성능 최적화

* 이미지 압축 및 WebP 사용 권장
* 지연 로딩(lazy loading) 적용
* 폰트 수 최소화
* 불필요한 애니메이션 과다 사용 지양

### 6.6 접근성 및 사용성

* 버튼 크기는 모바일에서 누르기 쉽게 설계
* 텍스트 대비 확보
* 지나치게 작은 글씨 사용 금지
* 배경음악이 있다면 자동재생보다 사용자 선택 방식 권장

### 6.7 SEO 및 메타 정보

* 제목(title), 설명(description), 썸네일 이미지 설정
* 카카오톡/문자 공유 시 미리보기 품질 확보

### 6.8 운영 편의성

* 사진/문구/날짜/주소를 한 파일에서 쉽게 수정 가능하도록 구조화
* 예: `src/data/wedding.ts` 또는 `src/content/site.ts`
* 하드코딩 분산을 줄여 AI 수정 요청 시 효율 개선

## 11. 개발 시 주의사항

### GitHub Pages 제약

* 서버 사이드 기능 없음
* 정적 사이트 중심으로 설계해야 함
* API 키 노출 방식 검토 필요

### 카카오지도 API 주의

* 도메인 등록 필요 여부 확인
* GitHub Pages 배포 주소 기준으로 설정 검토
* 공개 가능한 키와 보안상 민감한 키 구분 필요

### 이미지 관리 주의

* 고화질 원본을 그대로 올리면 모바일 로딩이 느려질 수 있음
* 업로드 전 리사이즈/압축 권장

### YouTube 임베드 주의

* 과도한 자동재생/소리 재생은 모바일 UX를 해칠 수 있음
